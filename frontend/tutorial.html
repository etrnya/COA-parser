🔬 COA 解析助手 使用指南

歡迎使用 AI COA 證書解析與驗證中心。請按照此指南申請免費的 Gemini API 金鑰、設定 Google 試算表（GAS）團隊雲端同步，並熟悉高效率的人工校對驗證工作流。

🔑 步驟一：取得免費 Google Gemini API 金鑰

本軟體使用 Google Gemini 多模態視覺 API 在本機解析 PDF 或圖片格式的 COA 證書。請按照以下步驟申請金鑰：

  1. 前往 Google AI Studio 首頁。
  2. 登入您的 Google 帳號。
  3. 點擊左上角的藍色 "Get API key" 按鈕。
  4. 選擇專案（或新增專案），然後點擊 "Create API Key in existing project"。
  5. 複製生成的金鑰，回到本軟體，進入 「系統設定面板」，將金鑰貼上至 「Gemini API 金鑰」 輸入框，然後點擊右側的 「儲存並測試」。

🔒 安全性聲明

您的 API 金鑰已透過作業系統內建的憑證庫（Windows 認證管理員/Keyring）進行加密儲存。金鑰絕不會傳送到 Google 官方 AI API 端點之外的任何第三方伺服器，安全無虞。

📊 步驟二：設定 Google 試算表 (GAS) 團隊共享同步

若要啟用團隊共同協作（多人共享已經驗證過的 COA 快取、節省 AI 代幣消耗），可以透過 Google Apps Script (GAS) 串接雲端試算表：

  1. 在您的 Google 雲端硬碟建立一個新的「Google 試算表」。
  2. 打開該試算表，從上方選單中選擇 「擴充功能」 ➔ 「Apps Script」。
  3. 刪除程式碼編輯器內的原有內容，將下方提供的 **GAS 範本程式碼** 完整複製貼上進去。
  4. 點擊上方 💾 「儲存專案」。
  5. 點擊右上角的 「部署」 ➔ 「新增部署」。
  6. 點擊左側設定齒輪，選取類型為 「網頁應用程式」 (Web app)。
  7. 進行部署設定：
    • 將應用程式執行為： 選擇「我」（您的 Google 帳號）。
    • 誰有權限存取： 選擇「任何人」(Anyone)（此設定用以允許桌面端進行 API 加密連線寫入）。
  8. 點擊 「部署」，接著點擊 「授予存取權限」 並登入 Google 帳號授權寫入。
  9. 部署完成後，複製產生的 「網頁應用程式網址」 (結尾通常為 /exec)。
  10. 回到本軟體的 「系統設定面板」，將網址貼入 「Google Apps Script (GAS) 串接網址」 輸入框，然後點擊下方 「儲存設定並套用」。

💡 團隊協作快取機制 (節省 API 成本)

當設定完畢後，只要有任何同事在人工驗證中心儲存了一份 COA 證書，該檔案的雜湊值與驗證數據就會自動存入 Google 試算表。當其他同事掃描相同檔案或資料夾時，系統會優先比對試算表，若匹配到快取，將會「瞬間」載入數據，完全不耗費任何 Gemini API 金鑰額度！

Google Apps Script (GAS) 網頁端範本程式碼 
function doGet(e) {
  var sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
  var action = e.parameter.action;
  
  if (action === "check_cache") {
    var hash = e.parameter.hash;
    var data = sheet.getDataRange().getValues();
    
    for (var i = 1; i < data.length; i++) {
      if (data[i][2] === hash) {
        var result = {
          product_raw: data[i][3],
          product_std: data[i][4],
          brand_raw: data[i][5],
          brand_std: data[i][6],
          batch_no: data[i][7],
          expiry_raw: data[i][8],
          expiry_std: data[i][9],
          purity_raw: data[i][10],
          purity_std: data[i][11],
          amount_raw: data[i][12],
          amount_std: data[i][13],
          cas_no_raw: data[i][14],
          cas_no_std: data[i][15],
          mw_raw: data[i][16],
          mw_std: data[i][17],
          storage_raw: data[i][18],
          storage_std: data[i][19]
        };
        return ContentService.createTextOutput(JSON.stringify({
          status: "success",
          found: true,
          data: result
        })).setMimeType(ContentService.MimeType.JSON);
      }
    }
    return ContentService.createTextOutput(JSON.stringify({
      status: "success",
      found: false
    })).setMimeType(ContentService.MimeType.JSON);
  }
  
  return ContentService.createTextOutput(JSON.stringify({
    status: "error",
    message: "Invalid GET action"
  })).setMimeType(ContentService.MimeType.JSON);
}

function doPost(e) {
  try {
    var sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
    
    if (sheet.getLastRow() === 0) {
      sheet.appendRow([
        "Timestamp", "File Name", "SHA256 Hash", 
        "Product Name (Original)", "Product Name (Standardized)", 
        "Brand (Original)", "Brand (Standardized)",
        "Batch No", 
        "Expiry Date (Original)", "Expiry Date (Standardized)", 
        "Purity (Original)", "Purity (Standardized)", 
        "Amount (Original)", "Amount (Standardized)",
        "CAS Number (Original)", "CAS Number (Standardized)",
        "Molecular Weight (Original)", "Molecular Weight (Standardized)",
        "Storage Condition (Original)", "Storage Condition (Standardized)"
      ]);
    }
    
    var postData = JSON.parse(e.postData.contents);
    
    if (postData.action === "save_entry") {
      var hash = postData.hash;
      var data = sheet.getDataRange().getValues();
      
      for (var i = 1; i < data.length; i++) {
        if (data[i][2] === hash) {
          var rowNum = i + 1;
          sheet.getRange(rowNum, 1, 1, 20).setValues([[
            new Date(),
            postData.file_name,
            hash,
            postData.product_raw,
            postData.product_std,
            postData.brand_raw,
            postData.brand_std,
            postData.batch_no,
            postData.expiry_raw,
            postData.expiry_std,
            postData.purity_raw,
            postData.purity_std,
            postData.amount_raw,
            postData.amount_std,
            postData.cas_no_raw,
            postData.cas_no_std,
            postData.mw_raw,
            postData.mw_std,
            postData.storage_raw,
            postData.storage_std
          ]]);
          return ContentService.createTextOutput(JSON.stringify({
            status: "success",
            message: "Entry updated successfully"
          })).setMimeType(ContentService.MimeType.JSON);
        }
      }
      
      sheet.appendRow([
        new Date(),
        postData.file_name,
        hash,
        postData.product_raw,
        postData.product_std,
        postData.brand_raw,
        postData.brand_std,
        postData.batch_no,
        postData.expiry_raw,
        postData.expiry_std,
        postData.purity_raw,
        postData.purity_std,
        postData.amount_raw,
        postData.amount_std,
        postData.cas_no_raw,
        postData.cas_no_std,
        postData.mw_raw,
        postData.mw_std,
        postData.storage_raw,
        postData.storage_std
      ]);
      
      return ContentService.createTextOutput(JSON.stringify({
        status: "success",
        message: "Entry appended successfully"
      })).setMimeType(ContentService.MimeType.JSON);
    }
  } catch (err) {
    return ContentService.createTextOutput(JSON.stringify({
      status: "error",
      message: err.toString()
    })).setMimeType(ContentService.MimeType.JSON);
  }
}

⌨️ 步驟三：快速鍵鍵盤確認流

為了在處理大量證書檔案（例如 50+ 張 COA）時將人工確認的吞吐量最大化，人工驗證中心 支援了純鍵盤快速鍵操作：

  • Ctrl + Enter : 儲存並繼續。自動儲存目前表單的修改值、將檔案在 SQLite 本機資料庫中標記為「已完成」、同步至雲端 Google Sheet，並「立刻」載入下一個待處理檔案。
  • Ctrl + ↓ (向下方向鍵) : 跳過檔案。直接跳到下一個檔案而不儲存任何修改。
  • Ctrl + ↑ (向上方向鍵) : 回到上一個。返回到隊列中的上一個檔案。

🔍 數據異常警告

凡是觸發規則引擎攔截的欄位（例如：有效期限已過期、純度不符合常規比例 0-100%、或者 Gemini 判斷置信度為「低 (low)」的欄位），驗證卡片下方都會亮起黃橘色的 「數據校正警告」。在審核這些檔案時請特別留意對照原文！

📈 步驟四：匯出 Excel 設定自訂

在點擊右上角的「匯出 Excel 報告」之前，您可以在 「系統設定面板」 中針對 Excel 表單欄位自訂：

  • 拖曳調整順序：以滑鼠按住三條線圖示 ☰ 上下拖曳，即可調整 Excel 匯出後的列順序，完美契合實驗室既有的登錄表。
  • 欄位顯示開關：取消勾選核取方塊可以將不需要的欄位隱藏，不匯出至 Excel 報表中。
  • 專業報表美化：產生的 summary.xlsx 已內建欄寬自動貼合、首行凍結（啟用篩選器）、以及深藍色表頭與粗體文字設計，可直接複製貼入實驗室日常工作記錄！
